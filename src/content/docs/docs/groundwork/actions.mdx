Each action is a focused step you call when you need it. They're **stateless**: every action reads `.groundwork.json` first, does its thing, and writes atomically inside fences. `init` is the only one that doesn't need an existing plan folder.

| Action | What it does |
| --- | --- |
| **init** | First-run interview (goal + profile) and scaffolds the spine + `.groundwork.json`. |
| **research** | Spawns a researcher agent that fills `02`/`03` in place ‚Äî external (cited URLs) and internal (`path:line`). `--sweep` fans out finders by angle via a Workflow. |
| **design** | Produces ‚â•2 comparable `designs/*.html` for the current phase and locks one. Profile-gated. |
| **subplan** | Scaffolds a focused `NN-*.md` from one of three archetypes ‚Äî **diff-plan**, **decision-doc**, **bug-doc**. |
| **review** | The highest-value recurring action. A reviewer agent appends a new dated **Round** to `04`, allocates gap IDs, and re-syncs affected docs by ID. `--panel` runs a verified, adversarial reviewer panel. |
| **clarify** | Readiness gate ‚Äî scans for open questions, unspecified IDs, and missing locked designs before orchestrate. |
| **orchestrate** | Generates `09-orchestration.md` (waves, gates, per-task briefs) from `05`. `--emit-workflow` also writes a runnable Workflow ‚Äî see [Orchestrate & Workflow](/docs/groundwork/orchestrate). |
| **refresh-board** | (Re)generates `artifact/board.html` from the current docs. |
| **explorer** | (Re)generates `artifact/explorer.html` ‚Äî a fully-offline file tree + tabbed viewer with search. See [The explorer](/docs/groundwork/explorer). |
| **plans-index** | (Re)generates `<plans-dir>/_index.html` ‚Äî a cross-plan dashboard, one card per plan with drill-in to each explorer / board. |
| **status** | Read-only freshness + ID + design-coverage report. Safe to run anytime. |

(`refresh-living-spec` regenerates the auto tabs of `artifact/index.html` and is there when you need it.)

## The core loop

A typical plan moves **init ‚Üí research ‚Üí review ‚Üí orchestrate**, with `review` re-run whenever the plan changes. Review is where a plan gets materially better: it catches the tracking file silently drifting from the plan, surfaces risks, and folds them back in by ID ‚Äî deterministically, not by guessing.

Three actions can opt into a [Workflow](/docs/groundwork/orchestrate) fan-out (`research --sweep`, `review --panel`, `orchestrate --emit-workflow`) ‚Äî never by default, because it's heavyweight and billed. Drop the flag and behaviour is identical to the single-agent path.
