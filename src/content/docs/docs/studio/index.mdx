**Studio** (`com.ikenga.studio`) is the orchestration spine for AI film. It turns a script into a finished cut on one board — breakdown, cast & world, storyboard, generation, and stitch — and conducts the generators you already use rather than replacing them.

> Higgsfield, Flow, and fal make shots. Ikenga makes the film.

<Aside type="caution" title="Coming to the registry">
Studio runs in the Ikenga shell today and a real master MP4 has shipped end-to-end from it, but it is not yet published to the registry — there's no `ikenga add` command for it yet. The interface screens in these docs are the locked UI **concepts**, shown on the “The Forge” demo film. See the [package page](/packages/studio) for status.
</Aside>

## The problem it removes

Every AI generator is an island. Between them, the actual film — the through-line, the consistency, the record of what made each frame — falls on the floor: your cast and world drift, prompts rot in spreadsheets, no clip remembers which model and seed made it, and any change means re-stitching by hand.

Studio removes that by being the **spine**: one place for the script, the cast, the locations, the prompts, the clips, and the cut, holding identity and provenance so nothing drifts and nothing is lost.

## The pipeline

One continuous board carries the work from the first line of script to the delivery master:

```text
Script → Breakdown → Refs & Cast → Storyboard → Generate → Stitch & Edit
```

## Two tracks

Generation runs on two tracks that both land on the same shot:

- **Track A — Studio generates.** Studio calls the generator directly through **fal.ai** — seed-locked, cost-tracked. Best for stills, character & location plates, and fast previews (FLUX, Kling, Veo, Wan, Hailuo, LTX).
- **Track B — you direct.** For web-only tools with no public API, Studio hands you a shaped per-shot prompt package; you generate in Higgsfield, Flow, Sora, or Midjourney and drop the clip back onto the shot.

## Plan it in Groundwork, shoot it in Studio

Studio owns execution — the shot board, render state, approvals, exports. Pre-production lives one step upstream in Groundwork's [`film`](/docs/groundwork/profiles/) profile: logline, treatment, cast bible, lookbook, shotlist, budget. The [idea-to-cut guide](/docs/groundwork/film-workflow/) walks the whole loop, and [Concepts](/docs/studio/concepts/) covers each surface in the app.

## Agent-drivable

Studio exposes a ~40-tool MCP surface, so the whole board — creating anchors, importing a breakdown, rendering shots, approving, composing exports — can be driven by an agent as well as by hand.
