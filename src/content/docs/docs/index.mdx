**Ikenga** is your personal AI-augmented workspace — a seat of strength brought to life through consecration, addressed daily, holding what your right hand has earned.

It's a desktop shell that hosts independently installable mini-apps (**pkgs**), driven by a pluggable AI engine (your **Chi**), and controllable from the terminal via the **Iyke** CLI. The shell itself is industry-neutral; verticals enter through pkgs and through scaffolding presets.

A short glossary you'll meet again across these docs:

- **Ikenga** — the workspace itself. A personal seat of strength.
- **Obi** — the home of your workspace; the shell that holds your panes.
- **Chi** — the AI engine that drives your workspace. One Chi at a time (Claude Code, Codex, Gemini, …).
- **Iyke** — the runtime CLI that lets any terminal or MCP client drive a running Obi.
- **Consecration** — the first-run ritual that brings a new Ikenga to life.

This documentation covers:

- **Pkgs** — the manifest-driven extension system that powers every mini-app.
- **Engines** — pluggable Chi adapters; the default ships as a pkg so it updates independently of the shell.
- **mcp-iyke** — the MCP server that exposes a running Obi as tools so any model can drive it.
